Animation start & continuation — audit

Branch: task/runtime-editor-port-alignment · 2026-05-18

Frame: MODEL.md (local slot-phase coordination, no global tick) + feedback_run_is_input_only (bodies self-poll via RAF, call wire.load) + project_runstart_concept_needed (two known mount hacks: seed-as-prefill, InputBody self-start).

Bottom line

The substrate’s locality (per-body RAF + slot-phase polling + wire.load) is not a hack — it’s the design. The hacks concentrate at one seam: getting the first value into the system. Three places exploit React mount to inject a value with no producer. All three collapse into one mechanism if the substrate gains a real run-start concept.

RECOMMENDED Suggestion 1 — slots can be born filled

Add an optional initialValue to the slot spec. At substrate construction, any slot with initialValue starts in slotPhase === "filled" carrying that value. Running and starting become the same rule applied to a slot that happens to already be filled. Applies to hack sites #1 (Wire seed) and #3 (ChainInhibitor heldRef) only — site #2 (InputBody RAF) was withdrawn as not a hack.

Consequences

  • Wire seed → declared on the edge as “dest slot born filled with X.” The useEffect + seededRef in Wire.tsx goes away.
  • ChainInhibitor heldRef → the held slot is born filled with seed. No parallel useRef.
  • InputBody self-start → not a hack; no change needed. Self-RAF on mount is the standard substrate pattern (identical to RelayBody/JoinBody); precondition is local canAccept + queue non-empty. Site #2 was withdrawn.
  • topogen finding (B) disappears. topogen generates ch <- initialValue from the slot spec, the same way substrate-r honors it. One concept, both runtimes.

Suggestion 2 — explicit Seed node kind

Make the seed an actual producer — a zero-input node whose only job is to emit once into its output wire at construction. Every initial value becomes a real wire event with a real source node.

Verdict. Take Suggestion 1. It promotes the unstated assumption (“things can have initial state”) into the model where it can be seen, edited, and honored by both runtimes. Suggestion 2 is purer but pays in topology bloat and doesn’t help topogen.
