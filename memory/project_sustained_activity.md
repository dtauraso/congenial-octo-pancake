Elementary particles are modeled as already-running structures with binding (AND gates) already part of the structure. They don't need external input to start — they self-cycle through their hierarchy of values continuously. Wavefunction amplitude sequences are sequences of number hierarchies.

**Two modes, same machinery:**
- **Self-sustaining mode:** Partitions cycle through their hierarchical data, AND gates bind hierarchies when they detect same timing. This is the particle at rest — already running, already bound.
- **Disruption mode:** External input perturbs the running system. The disruption causes data cascades, contrast changes (XOR), attention conflicts resolved by lateral inhibition. This is interaction.

**Key realization:** The current architecture (cascade inhibitors, AND gate binding, contrast detection, lateral inhibition) is the universal substrate used in both modes. It's not just a pipeline for processing input — it's the same machinery that sustains the particle's self-activity AND handles disruption from external input.

**Build order:** Get the disruption/response machinery solid first (current work), then later wrap it in the self-sustaining cycling layer. The response system needs to be working before you can meaningfully test perturbation of a running structure.

**Why:** Particles are self-contained oscillating structures. "Input" is not what starts processing — it disrupts an ongoing process. The attention conflict between internal cycling and external disruption is where contrast detection, inhibition, and cascading become meaningful.

**How to apply:** The current architecture is correct and shared by both modes. Don't redesign for self-sustaining activity — continue building the current machinery, then add the cycling/self-start layer on top when ready.
