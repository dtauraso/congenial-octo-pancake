Firing-Rule Fix Spec

Branch: task/firing-rule-guards · Source: docs/planning/visual-editor/firing-rule-fix-spec.md · Drafted 2026-05-18

Background

Logic audit (2026-05-18) surfaced a value-loss bug class in substrate-r/node-kinds.tsx. Five node bodies call node.consume(slotId) without first checking slotPhase === "filled" and/or outWire.canAccept. When the destination wire is in-flight, wire.load silently no-ops and the consumed value is dropped.

Affected bodies

  • High RelayBody
  • High RegisterBody
  • High InhibitRightGateBody
  • High JoinBody
  • Medium ChainInhibitorBody (inhibitWire branch)
  • Reference ReadGateBody — guards correctly

MODEL.md states "firing is precondition-gated" but does not explicitly mandate the slot-phase + canAccept guards before each consume. So this is a missing-contract issue as much as a per-body bug.

Contract statement (MODEL.md addition)

Append to the "Firing rule and slot writes" section, after the paragraph ending "...RAF pacing is the observation window, not an independent clock competing for authority":

Output-readiness precondition. Before consuming any slot, a node body must verify two conditions locally: (a) every input slot it intends to consume is in filled phase, and (b) every destination wire it intends to load reports canAccept === true. Both checks are read-only observations of local state — no signal crosses to another node. If either condition is unmet, the body returns without consuming anything and re-observes on the next poll frame. A body that consumes a slot before verifying canAccept loses the value silently when the wire no-ops the load; this is a contract violation, not a retry. The precondition is all-or-nothing: partial consumption is not permitted.

No banned vocabulary introduced. canAccept is a pure read of local wire and dest-slot phase; the check stays local to the observing body.

Mechanism

Each body keeps its own inline guards, following ReadGateBody as the reference. The only shared surface needed is the already-existing wire.canAccept getter and node.slotPhase(). No new primitive.

MODEL.md says "a node's identity is its firing rule" — the rule stays on the body.

A tryFire(reads, writes, action) helper would need to accept the set of input slots and output wires as arguments and call action only when both preconditions hold. That signature is a firing rule — it would become a policy engine that has to encode per-body distinctions:

  • ReadGate passes slots[0] through
  • Join concatenates both
  • ChainInhibitor swaps a held ref

Encoding those into a shared helper turns it into a coordinator-shaped abstraction in a medium wrapper. Rejected on substance-vs-medium grounds.

Per-body edits

File: tools/topology-vscode/src/webview/substrate-r/node-kinds.tsx. Switch each card to compare current vs. proposed shape.

RelayBody (lines ~141–147)

const value = node.consume(slotId);
wire.load(value);
if (node.slotPhase(slotId) !== "filled") return;
if (!wire.canAccept) return;
const value = node.consume(slotId);
wire.load(value);

RegisterBody (lines ~258–267)

const incoming = node.consume(slotId);
const emitted = heldRef.current;
heldRef.current = incoming;
wire.load(emitted);
if (node.slotPhase(slotId) !== "filled") return;
if (!wire.canAccept) return;
const incoming = node.consume(slotId);
const emitted = heldRef.current;
heldRef.current = incoming;
wire.load(emitted);

Held-swap must only happen when both preconditions pass — otherwise consume-without-load corrupts heldRef.

InhibitRightGateBody (lines ~31–52)

const leftValue = node.consume(leftSlotId);
node.consume(rightSlotId);
// ...
wire.load(leftValue); // unguarded
const leftFilled  = node.slotPhase(leftSlotId)  === "filled";
const rightFilled = node.slotPhase(rightSlotId) === "filled";
if (!leftFilled && !rightFilled) return;
if (leftFilled && !rightFilled && !wire.canAccept) return;
const leftValue = node.consume(leftSlotId);
node.consume(rightSlotId);
if (leftFilled && !rightFilled) wire.load(leftValue);

Only the firing branch needs canAccept; the inhibit-consume branch produces no output.

JoinBody (lines ~168–175)

const va = node.consume(slotAId);
const vb = node.consume(slotBId);
wire.load([va, vb]);
if (node.slotPhase(slotAId) !== "filled") return;
if (node.slotPhase(slotBId) !== "filled") return;
if (!wire.canAccept) return;
const va = node.consume(slotAId);
const vb = node.consume(slotBId);
wire.load([va, vb]);

Node.fill mis-wiring

Decision Keep the silent no-op.

Node.fill (Node.tsx:64) silently returns when s.phase !== "empty". This is the deferred-deliver safety path: Wire.tsx deliverIfPending retries fill across RAF frames until the slot empties. Throwing here would break that path.

The "mis-wiring caught at parseSpec" guarantee belongs at the spec boundary (two wires assigned the same (node, slotId) pair), not at fill time.

Optional: add if (traceId) postLog("trace.fill.deferred", …) inside the early-return so the .probe log surfaces repeated deferrals as a signal, not an error.
