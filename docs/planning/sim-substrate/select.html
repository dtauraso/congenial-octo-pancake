Select — visible-state snapshot

A goroutine is parked at a select. Each case is one chan op (send or recv) plus a readiness predicate. A case is ready when its op would not block: send-case ready ⇔ the chan has a free slot or a waiting receiver; recv-case ready ⇔ the chan has a buffered item or a waiting sender. default, if present, is always ready and fires only when no other case is ready.

Visible-state firing rule: collect all ready cases. If the set is non-empty and contains non-default cases, pick the lowest-index ready non-default case (this sketch uses lowest-index for determinism — Go's runtime randomises; the rebuild plan will pick its own rule). If only default is ready, fire default. If the set is empty, the goroutine remains parked.
