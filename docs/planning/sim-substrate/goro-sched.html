Goroutine + Scheduler — visible-state snapshot

One CPU slot (P), a FIFO run queue, and a blocked set. Each goroutine is in exactly one of three places: runnable (waiting in the queue), running (in the CPU slot, with a quantum remaining), or blocked (parked on a reason — chan-send, chan-recv, etc.). Each click is a discrete tick. No hidden state: if it isn't drawn, it doesn't exist.

The scheduler rule (visible-state transition): if the CPU slot is empty and the queue is non-empty, dequeue head into the slot with a fresh quantum. The running goroutine's quantum decrements each tick; at zero it is preempted back to the queue tail. park moves the running goroutine to the blocked set. unblock moves a chosen blocked goroutine to the queue tail.
